Worktrees let you run multiple local agents at the same time without them editing the same folder.

Normally, your repo is one folder on your computer. If you ask Codex, Claude Code, or Cursor to all work in that same folder, the changes can get mixed together.

A worktree is a second local copy of the repo for a specific task. It still uses the same Github repo, but it has its own files and branch.

The most useful pattern is:

1. Create a worktree for one small task
2. Let the agent make changes there
3. Review the diff
4. Commit and push that branch
5. Open a PR from the worktree

For this exercise, you should try to create two changes at the same time to see the agents multitask. For example:

- Improve one empty state
- Add a new component
- Fix confusing copy
- Add a loading state
- Update one component
- Add one missing test

Avoid auth, billing, database migrations, or big redesigns for your first worktree.

## Claude Code
By default, Claude Code will have worktrees enabled.

## Codex

Codex requires you to select worktrees when you want to use them.
![[CleanShot 2026-05-03 at 13.52.41.png]]

Once your change is ready, you can convert the workree to a branch.
![[CleanShot 2026-05-03 at 13.56.53.png]]
